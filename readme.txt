Комментарии к заданию 3:

Для правильной выгрузки данных из исходных файлов, их следовало отформатировать.
Файл format.py обрабатывает исходные файлы <name>.txt и сохраняет их под названием <name>2.txt,
которые в последствии и выгружаются в базу данных инструкцией bulk insert


Исходные данные третьего задания имели некорректные строки:

1) В файле users.txt присутствует множество строк с равными уникальными идентификаторами (id),
   что не позволяло сделать этот атрибут первичным ключом

2) В файле quality.txt присутствуют внешние ключи, отсутствующие в файле lessons.txt,
   что не позволяло загрузить такие данные с ограничением на внешний ключ, отсылающийся на lessons


Для решения первой проблемы при форматировании в format.py из исходного файла были удалены полные дубликаты.

Для решения второй проблемы при заполнении инструкцией bulk insert были проигнорированы ограничения. 
Т.к. при использовании JOIN некорректные данные не попадут в выборку, эти данные не помешают решению поставленной задачи.
